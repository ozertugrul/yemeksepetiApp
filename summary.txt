============================================================
1) PROJE AMACI VE KAPSAM
============================================================
Bu proje, çok rollü bir yemek sipariş ekosistemi sunar:
- Mobil istemci: iOS (SwiftUI)
- Backend API: FastAPI (Python)
- Kimlik doğrulama: Firebase Authentication (ID Token)
- Uygulama verisi: PostgreSQL (Supabase)
- Opsiyonel öneri sistemi: pgvector + sentence-transformers

Ana hedef:
- Son kullanıcıların restoran gezmesi, sepete eklemesi ve sipariş vermesi
- Mağaza sahiplerinin restoran/menü/sipariş yönetmesi
- Adminin kullanıcı, restoran ve istatistik yönetimi yapması

3) YÜKSEK SEVİYE MİMARİ
============================================================
3.1 Mobil mimari
Entry point:
- YemeksepetiAppEntryPoint.swift
  - FirebaseApp.configure()
  - Tek AppViewModel oluşturur ve MainView’e verir

Orkestrasyon:
- AppViewModel
  - authService, cart, dataService, orderService, couponService içerir
  - Admin işlemleri için AdminAPIService’e delegasyon yapar
  - Child objectWillChange sinyallerini üst seviyeye taşır

Ağ katmanı:
- APIClient (actor)
  - Tüm HTTP isteklerinde Firebase ID token ekler
  - 401 -> apiUnauthorized notification
  - 403 -> apiForbidden notification
- Diğer servisler APIClient üzerinden endpoint bazlı çalışır

Oturum:
- AuthService
  - Firebase auth state listener
  - /users/me ile profil/rol senkronu
  - 401 aldığında signOut (anlık kick davranışı)
  - 403 aldığında refreshCurrentUser (rol değişimini çekme)

3.2 Backend mimari
- main.py
  - Router’ları /api/v1 altında toplar
  - CORS açık
  - lifespan ile startup/shutdown logları
- core/
  - config.py: env tabanlı ayarlar
  - database.py: async engine + session
  - auth.py: Firebase token + rol bazlı dependency
- routers/
  - users, restaurants, orders, admin, recommendations
- repositories/
  - SQL repo katmanı (CRUD + sorgular)
- models/
  - SQLAlchemy ORM modelleri
- schemas/
  - Pydantic request/response şemaları (CamelModel)

  - DB’den rolü okur
  - DB’de user yoksa 401 döner

Kritik güvenlik davranışları:
- Silinen/rolü düşen kullanıcı için canlı oturumun yetki kaybı:
  - 401 -> mobil signOut

12) GENEL DEĞERLENDİRME
============================================================
Bu proje, MVP seviyesini geçmiş, gerçek role-based operasyonları olan hibrit bir yemek
sipariş platformudur. Mimari ayrım (iOS view/service + FastAPI router/repository) net,
özellik kapsamı geniş ve admin/owner tarafı işlevseldir.

En güçlü taraflar:
- Çok rollü iş akışı
- Gerçek API tabanlı veri modeli
- Admin operasyonları
- Güvenlik/oturum davranışlarında son dönemde yapılan sertleştirmeler

En büyük geliştirme fırsatları:
- Kupon ve ödeme altyapısının tam backend desteğine taşınması
- Polling yerine gerçek zamanlı kanal
- Büyük view dosyalarının modüler parçalanması
- Endpoint yetki testlerinin sistematik hale getirilmesi
